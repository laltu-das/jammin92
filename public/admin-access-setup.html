Admin Access Code Setup

Secure your admin panel with an additional access code

Security Notice

This admin access code adds an extra layer of security to prevent unauthorized access to your admin panel.

Features

  • Additional security layer beyond email/password
  • Code stored securely in .env file (not in database)
  • Prevents unauthorized access even if user credentials are compromised
  • Easy to change and manage
  • Works with existing admin authentication system
  • Required for both admin login AND registration
  • Prevents unauthorized admin account creation
1

Add Admin Access Code to .env File

Add the following line to your .env file:

ADMIN_ACCESS_CODE=your-secret-code-here

Important: Replace your-secret-code-here with a strong, unique code that only authorized admins know.

Tip: Use a combination of letters, numbers, and special characters. Example: Admin2024!Secure

4

Test Admin Login

  1. Visit your admin login page: /admin/login
  2. Enter your admin email and password
  3. Enter the admin access code you set in the .env file
  4. Click "Login"
  5. You should be successfully redirected to the admin dashboard
Note: If the access code is incorrect, you'll see an error message: "Invalid admin access code."
5

Test Admin Registration

  1. Visit your admin registration page: /admin/register
  2. Enter your name, email, and password
  3. Confirm your password
  4. Enter the admin access code you set in the .env file
  5. Click "Register Account"
  6. You should be successfully redirected to the admin dashboard
Important: Without the correct admin access code, no one can create new admin accounts!
!

Troubleshooting

Common Issues
  • Code not working: Clear your cache and restart the server
  • Still getting access denied: Verify the .env file syntax
  • Forgot the code: Check your .env file or generate a new one
Commands
# Clear cache
php artisan cache:clear
php artisan config:clear

Security Best Practices

  • Never share your admin access code with anyone
  • Change the access code regularly
  • Don't use common words or predictable patterns
  • Keep your .env file secure and never commit it to version control
  • Only share the code with trusted administrators
